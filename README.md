# Documentación Meembly

Guías operacionales para el equipo comercial de **Meembly** — la plataforma de fidelización para clubes deportivos.

Si acabas de llegar, empieza aquí:

## Nueva incorporación

1. **[onboarding-comercial.md](./onboarding-comercial.md)** — tu primera semana paso a paso. Léelo entero el día 1, te lleva 30-40 min. Cubre producto, setup técnico, flujo del lead, pitch + objeciones y contactos de escalado.

## Diagramas

Las guías usan diagramas **mermaid** que se renderizan como imagen en GitHub. Si las abres desde el navegador (URL del repo), lo ves bonito. En VSCode "preview markdown" también funciona.

Para exportar a PDF: abre la URL en Chrome/Safari → Cmd+P → "Guardar como PDF". Mermaid se imprime renderizado.

## ¿Qué NO está aquí?

- Guías internas de Owner (Samuel) y Admin (Marta) — contienen flujos de gestión del equipo, permisos, configuración sensible. Si necesitas saber cómo se hace algo que compete a esos roles, pregunta directamente.
- Capturas reales de la app — pendientes, listadas en [`screenshots/README.md`](./screenshots/README.md).

## Cómo proponer cambios

Si detectas algo desactualizado o mal explicado tras tu experiencia real, abre un issue o un PR directo. Estas guías se afinan con cada comercial nuevo que entra.
